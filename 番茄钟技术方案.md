# 番茄钟技术方案

## 目标

基于《番茄钟逻辑.md》的业务描述，开发一个纯前端番茄钟页面。页面不依赖后端服务，核心能力包括任务输入、倒计时圆环、专注/短休息/长休息自动流转、暂停、重置、跳过、提示音、语音播报和本地状态保存。

## 技术栈

推荐使用：

- Vite
- React
- TypeScript
- CSS Modules 或普通 CSS
- lucide-react
- localStorage
- Web Audio API
- SpeechSynthesis API
- SVG circle

如果希望保持最轻量，也可以使用 `index.html + style.css + app.js` 实现。不过考虑到状态逻辑会逐渐复杂，推荐使用 `Vite + React + TypeScript`。

## 页面结构

页面整体分为三块：顶部任务区、中部计时器区、底部控制区。

### 顶部任务区

顶部展示一个单行文本输入框。

- placeholder：`当前要做的事情...`
- 用户可以随时输入或修改当前任务
- 输入内容实时同步到计时器下方
- 当前任务可保存到 `localStorage`

### 中部计时器区

中部是页面视觉核心。

- 使用 SVG 圆环展示倒计时进度
- 圆环中心显示剩余时间，例如 `25:00`
- 圆环下方显示当前状态
- 状态下方显示当前任务名称

状态示例：

- 准备专注
- 专注中
- 短休息
- 长休息
- 已暂停

不同阶段使用不同视觉反馈：

- 专注：番茄红 / 橙红，浅红背景
- 短休息：薄荷绿 / 青绿色，浅绿背景
- 长休息：深蓝 / 靛蓝，浅蓝背景

### 底部控制区

底部展示设置和控制按钮。

- 连续专注轮数选择器：`1-8`，默认 `4`
- 重置按钮
- 开始 / 暂停 / 继续主按钮
- 跳过按钮

主按钮建议使用大圆形按钮，重置和跳过使用图标按钮。

## 核心状态设计

```ts
type Phase = 'focus' | 'shortBreak' | 'longBreak';

type TimerState = {
  currentTask: string;
  phase: Phase;
  remainingSeconds: number;
  completedRounds: number;
  targetRounds: number;
  isRunning: boolean;
};
```

固定时长：

```ts
const DURATIONS = {
  focus: 25 * 60,
  shortBreak: 5 * 60,
  longBreak: 15 * 60,
};
```

## 状态流转规则

### 初始状态

- `phase = 'focus'`
- `remainingSeconds = 25 * 60`
- `completedRounds = 0`
- `targetRounds = 4`
- `isRunning = false`
- `currentTask = ''`

页面显示 `25:00`，状态为“准备专注”。

### 开始专注

用户点击开始按钮后：

- 如果当前任务为空，提示“请先输入当前任务”
- 如果任务不为空，开始倒计时
- 状态显示为“专注中”
- 浏览器标题同步显示当前剩余时间

### 专注结束

专注倒计时归零后：

```ts
completedRounds += 1;

if (completedRounds >= targetRounds) {
  phase = 'longBreak';
  remainingSeconds = 15 * 60;
} else {
  phase = 'shortBreak';
  remainingSeconds = 5 * 60;
}

isRunning = true;
```

同时播放提示音并语音播报：

```txt
专注时间结束，休息一下吧
```

### 短休息结束

短休息倒计时归零后：

```ts
phase = 'focus';
remainingSeconds = 25 * 60;
isRunning = true;
```

同时播放提示音并语音播报：

```txt
休息结束，继续加油
```

### 长休息结束

长休息倒计时归零后：

```ts
completedRounds = 0;
phase = 'focus';
remainingSeconds = 25 * 60;
isRunning = true;
```

同时播放提示音并语音播报：

```txt
长休息结束，新一轮番茄钟即将开始
```

### 暂停与继续

运行中点击主按钮：

```ts
isRunning = false;
```

暂停后再次点击主按钮：

```ts
isRunning = true;
```

暂停不会改变当前阶段，也不会重置剩余时间。

### 重置

点击重置按钮：

```ts
remainingSeconds = DURATIONS[phase];
isRunning = false;
```

重置只回到当前阶段的初始时间，不改变 `completedRounds`。

### 跳过

点击跳过按钮时，直接执行当前阶段结束逻辑：

```ts
handlePhaseComplete();
```

跳过应按正常流转规则进入下一个阶段。

## 计时实现方案

不建议只使用简单的 `setInterval(() => remainingSeconds - 1)`，因为浏览器后台标签页或页面卡顿时会造成倒计时误差。

推荐使用时间戳校准：

```ts
const endAt = Date.now() + remainingSeconds * 1000;
```

运行中每隔 250ms 或 1000ms 计算剩余时间：

```ts
remainingSeconds = Math.max(0, Math.ceil((endAt - Date.now()) / 1000));
```

这样即使页面短暂卡顿，倒计时也能和真实时间保持一致。

## SVG 圆环进度

圆环进度基于当前阶段总时长和剩余时间计算。

```ts
const totalSeconds = DURATIONS[phase];
const progress = 1 - remainingSeconds / totalSeconds;
```

SVG 圆环可使用 `strokeDasharray` 和 `strokeDashoffset` 控制进度。

```ts
const circumference = 2 * Math.PI * radius;
const strokeDashoffset = circumference * (1 - progress);
```

## 浏览器标题同步

运行中根据倒计时状态更新网页标题：

```ts
document.title = `[${formatTime(remainingSeconds)}] ${phaseLabel}`;
```

暂停或初始状态可以显示：

```ts
document.title = '番茄钟';
```

## 提示音与语音播报

阶段结束时执行：

```ts
playBeep();
speak('专注时间结束，休息一下吧');
```

提示音可以使用 Web Audio API 生成短促提示音，也可以引入本地音频文件。

语音播报使用浏览器原生 `SpeechSynthesis API`。

注意：浏览器通常要求用户先与页面交互后才能播放音频或语音。因此建议在用户第一次点击“开始”时初始化音频上下文。

## 本地持久化

第一版建议保存：

```txt
currentTask
targetRounds
```

增强版可以继续保存：

```txt
completedRounds
phase
remainingSeconds
isRunning
lastUpdatedAt
```

如果保存运行中状态，页面刷新后可以根据 `lastUpdatedAt` 计算真实剩余时间，避免刷新导致计时不准确。

## 推荐目录结构

```txt
src/
  App.tsx
  components/
    TaskInput.tsx
    TimerCircle.tsx
    TimerStatus.tsx
    TimerControls.tsx
    RoundSelector.tsx
  hooks/
    usePomodoroTimer.ts
  utils/
    time.ts
    audio.ts
  styles/
    app.css
```

职责划分：

- `App.tsx`：页面整体组合
- `TaskInput.tsx`：任务输入
- `TimerCircle.tsx`：圆环进度和时间显示
- `TimerStatus.tsx`：阶段状态和当前任务展示
- `TimerControls.tsx`：开始、暂停、继续、重置、跳过
- `RoundSelector.tsx`：连续专注轮数选择
- `usePomodoroTimer.ts`：番茄钟核心状态机
- `time.ts`：时间格式化工具
- `audio.ts`：提示音和语音播报工具
